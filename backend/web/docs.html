Getting Started

Browse first, authenticate when you act.

Stoarama keeps stream discovery public so researchers can explore the catalog without setup. Sign in only when you need recording controls, clip downloads, API keys, or node management.

All video streams are captured at 10 frames per second. Image-poll streams capture individual frames on a configurable polling interval.

Core flow

How researchers typically work

  • Search the public stream catalog on /streams.
  • Open a stream detail page to inspect metadata and recorded clips.
  • Go to /recording when you want to start or monitor recording.
  • Use /account for session, API keys, and local recorder enrollment.
Auth model

What needs a signed-in account

  • Preparing authenticated clip downloads.
  • Turning recording on or off from the browser.
  • Creating or revoking API keys.
  • Enrolling and managing local recorder nodes.
API Reference

Public reads plus account-scoped actions.

The backend API stays unchanged. Use public dashboard endpoints for browsing and account-scoped endpoints when working with clips, keys, or browser-authenticated recording actions.

Public catalog

Search streams

curl "https://stoarama.example/api/v1/dashboard/streams?limit=25&q=seoul"

curl "https://stoarama.example/api/v1/dashboard/streams/15944"

curl "https://stoarama.example/api/v1/dashboard/recording/summary"
Authenticated clips

List and prepare downloads

curl -H "Authorization: Bearer <api-key>" \
  "https://stoarama.example/api/v1/account/streams/15944/clips?limit=25&offset=0"

curl -X POST "https://stoarama.example/api/v1/account/clips/download-prepare" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <api-key>" \
  -d '{"stream_id":15944,"segment_ids":[101,102,103]}'
Browser session

Recording actions from the UI

POST /api/v1/recording/streams/{id}/assign
POST /api/v1/recording/streams/{id}/unassign
GET  /api/v1/account/me
GET  /api/v1/account/api-keys
POST /api/v1/account/api-keys
Self-Serve Setup

Run local recording on your own machine.

Self-serve is the right path when researchers need local browser credentials while still using Stoarama’s catalog and account model.

Account setup

Start in account tools

  • Sign in with an email magic link.
  • Create or use an API key on /account.
  • Run stoarama recording youtube run --stream-id N from the machine with local credentials.
Operational split

What stays local vs remote

  • Cookies and local browser auth stay on your own machine.
  • Stoarama stores assignment state and recording metadata.
  • Researchers still browse streams and monitor recording in the main web UI.
