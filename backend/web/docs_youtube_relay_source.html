YouTube Relay Source

Use a Stoarama relay source on your own computer when a stream needs local YouTube cookies. The source machine resolves the stream locally, serves relay bytes, and Stoarama’s recording sinks pull from that relay.

What You Need

  • A Stoarama account and an enrolled yt_relay_source node.
  • ffmpeg and yt-dlp installed locally.
  • A public or private relay URL reachable from Stoarama’s sink servers.
  • YouTube cookies stored only on your own machine.

Install Flow

1. Create a node enrollment token from your account page.

2. Enroll your machine:

stoarama node enroll \
  --api-base-url https://stoarama.com \
  --token <enrollment-token> \
  --node-type yt_relay_source \
  --display-name "Mac mini relay"

3. Install the launchd service on macOS:

stoarama node yt-relay-source install-launchd \
  --public-base-url https://your-relay-host.example \
  --cookies-from-browser chrome

4. The service runs in the background and will show up on your account page with heartbeat status and available YouTube relay slots.

Foreground Run

If you want to test without installing launchd:

stoarama node yt-relay-source run \
  --public-base-url https://your-relay-host.example \
  --cookies-from-browser chrome

Important Rules

  • Cookies never leave your machine.
  • Stoarama only stores node state, route state, and relay metadata.
  • The recording sink stays remote; your source machine only resolves and relays.
