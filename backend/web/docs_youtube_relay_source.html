Local helper for restricted sources

Run a YouTube relay source on your own machine.

Use this when a stream needs browser cookies locally. Your machine resolves the source and relays bytes, while Sto-A-Rama keeps the catalog, assignment state, and recording sinks.

Cookies stay local Relay node belongs to your account Recording sink stays remote
Prereqs

What you need

  • A Sto-A-Rama account and an enrolled yt_relay_source node.
  • ffmpeg and yt-dlp installed locally.
  • A relay URL reachable from Sto-A-Rama sink servers.
  • YouTube cookies stored only on your own machine.
Account setup

Create a node enrollment token

Generate the token from your account page. The account page also shows the node once it starts heartbeating.

Open /account
Create a relay enrollment token
Copy the token once
Enroll the node on the source machine
Install flow

Enroll the machine

stoarama node enroll \
  --api-base-url https://stoarama.com \
  --token <enrollment-token> \
  --node-type yt_relay_source \
  --display-name "Mac mini relay"
macOS background service

Install launchd

stoarama node yt-relay-source install-launchd \
  --public-base-url https://your-relay-host.example \
  --cookies-from-browser chrome
Foreground testing

Run without launchd

stoarama node yt-relay-source run \
  --public-base-url https://your-relay-host.example \
  --cookies-from-browser chrome
Rules

Operational boundaries

  • Cookies never leave your machine.
  • Sto-A-Rama stores node state, route state, and relay metadata only.
  • The recording sink remains remote; the source machine resolves and relays.
