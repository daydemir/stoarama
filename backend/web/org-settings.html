How to set up your NAS (Synology)
  1. In DSM open Package Center and install Container Manager (skip if already installed).
  2. Create two shared folders: /volume1/stoarama-clips holds the downloaded footage (one subfolder per recording), and /volume1/stoarama-state holds only cursor.json, which records the last drained clip id so a restart resumes in place.
  3. In Container Manager > Project > Create, set the path to any empty folder (e.g. /volume1/docker/stoarama) and paste the docker-compose shown above as the source. Nothing else goes in that folder, there is no file to upload and nothing to build. It is pre-filled with your API base, your token, and your chosen poll interval.
  4. To validate read-only first, set STOARAMA_DRY_RUN to "1" (downloads and verifies clips, never releases them, cursor not persisted), then set it back to "0" for live operation.
  5. Start the project. restart: always keeps it running across reboots.
  6. Come back here and click Refresh. Within a minute or two the status turns GREEN / Connected.

Any Docker host: the same compose runs anywhere. Save it as docker-compose.yml and run docker compose up -d.

Copy the compose now. The token is shown once; use Rotate to get a new one.

If your NAS falls behind, clips left in managed storage over 24h bill at $0.10 per hour of footage, per month until the NAS catches up.

How to connect a computer (Mac or Linux)
  1. Enter a name above and click Connect a computer.
  2. Copy the install command that appears and run it in Terminal (Mac) or a shell (Linux). It installs the relay as a background service (launchd on Mac, systemd on Linux) that starts automatically on login.
  3. Public YouTube streams work right away, no setup needed. The relay resolves them through your own connection.
  4. To record private, members-only, or age-restricted YouTube, log into YouTube in Chrome on that computer, then run the command below in Terminal and click Always Allow when macOS asks for Keychain access. This exports only your YouTube and Google sign-in cookies, nothing else, and they never leave the machine. The installer offers this step automatically, and you can re-run it anytime.
  5. Come back here and click Refresh. Within a few seconds the computer shows Connected.

Windows support is coming. For now, Mac and Linux only.

Remove a computer / uninstall

Run this in Terminal (Mac) or a shell (Linux) on that computer. It stops and removes the relay background service and its files (~/.stoarama).

Also click Remove on the computer's row above to revoke its access here.
